## Stage 1 · Domain Research

### Pertanyaan kunci yang dijawab

**Q: Apa itu Pokmaswas dan kenapa jadi pivot user persona utama?**

Pokmaswas (Kelompok Masyarakat Pengawas) adalah kelompok masyarakat pesisir resmi binaan **Ditjen PSDKP — Kementerian Kelautan dan Perikanan**.

- Sudah punya **SK pengukuhan formal** (artinya: identitas legal sudah ada, tidak perlu KYC ulang)
- Terlatih dalam pengawasan sumber daya laut
- Tersebar **±2.000 kelompok aktif** di seluruh Indonesia
- Saat ini lapor manual via WhatsApp/kertas (pain point digitalisasi nyata)

**Q: Regulasi pesisir Indonesia yang relevan?**

- **PP 27/2021** — Penyelenggaraan Bidang Kelautan dan Perikanan
- **PP 5/2021** — Pelaksanaan Perizinan Berbasis Risiko (RZWP3K)
- **Permen KP 36/2023** — Pemberdayaan Pokmaswas
- **POJK 51/2017** — Penerapan Keuangan Berkelanjutan (untuk Corporate ESG)
- **UU PDP No. 27/2022** — Perlindungan Data Pribadi (relevan untuk donor data)
- **SRN-PPI** (Sistem Registri Nasional - PPI) — registry karbon nasional, tetap relevan tapi bukan primary value prop

**Q: Apa yang berubah dari positioning lama?**

| Sebelum | Sesudah |
|---|---|
| Carbon credit infrastructure | Coastal data integration platform |
| KLHK (PP 98/2021) sebagai regulator | KKP/PSDKP sebagai pembina utama |
| Mangrove only | Mangrove + abrasi + penyu + ekosistem pesisir |
| B2B focus (perusahaan beli kredit) | Multi-actor (Sahabat + Mitra + DKP + Corporate) |
| Pre-market carbon = niche | Public good integration = wider reach |

**Q: Kompetitor & lansekap?**

- **Indikator (BRGM)** — pemerintah, top-down monitoring mangrove
- **Tracart, Fairatmos** — private carbon registry, B2B-only
- **Asuh Pesisir, KKP One Data** — internal pemerintah
- **Gap yang ID-MAP isi:** bottom-up Pokmaswas + transparent donor + multi-stakeholder marketplace

## Stage 2 · Product Brief

### Vision Statement

> "Menjadi platform terpercaya yang mempercepat pemulihan dan keberlanjutan ekosistem pesisir nusantara melalui kolaborasi data, komunitas, dan teknologi."

### Mission Statement

> "Mengintegrasikan data dan tata kelola pesisir Indonesia melalui enam layanan inti — rehabilitasi mangrove, penyulaman, MRV digital, decarbonisasi aquaculture, perbaikan habitat penyu, dan pemberdayaan masyarakat pesisir — secara transparan, partisipatif, dan berkelanjutan."

### Target Users (Primary)

1. **Pokmaswas / Mitra** (B2B-G operator) — ±2.000 kelompok ber-SK PSDKP, fokus Jatim → Jateng → Sulsel → NTT
2. **Sahabat Pesisir** (B2C donor) — GenZ-Millennial 22-40 tahun di Jabodetabek-Surabaya-Bandung, donasi Rp 10rb-500rb
3. **DKP Provinsi/Kabupaten** (G2G audit) — perpanjangan tangan KKP di lapangan
4. **Korporasi ESG** (B2B sponsor) — emiten BEI wajib lapor POJK 51/2017
5. **NGO Konservasi** (mitra strategis) — YKAN, WWF, KEHATI

### Value Proposition Triangle

- Untuk **Pokmaswas**: alat kerja digital → laporan cepat → honor lebih cepat → pengakuan
- Untuk **Sahabat**: bukti realisasi donasi → sertifikat shareable → impact nyata
- Untuk **DKP**: dashboard agregat lintas kecamatan → laporan ke pusat lebih mudah

### 6 Layanan Inti (cores)

1. Rehabilitasi Mangrove
2. Penyulaman Mangrove
3. Jasa Pemantauan Monev (MRV) Mangrove
4. Decarbonisasi Aquaculture
5. Perbaikan Habitat Penyu
6. Pemberdayaan Masyarakat Pesisir

### Differentiator

- Bottom-up dari Pokmaswas, bukan top-down dari KLHK/BRGM
- Multi-actor marketplace, bukan registry tertutup
- Live MRV dengan QRIS donation loop yang langsung tersalurkan ke kelompok pelaksana
- Future-ready: data MRV kompatibel dengan SRN-PPI saat regulasi blue carbon matang

### FAQ External

**Q: Apa beda ID-MAP dengan platform donasi lingkungan lain?**
ID-MAP fokus eksklusif pada ekosistem pesisir Indonesia dan terhubung langsung ke Pokmaswas resmi binaan KKP. Setiap proyek divalidasi DKP daerah dan dapat diaudit publik.

**Q: Bagaimana donasi saya tersalurkan?**
100% dana donasi masuk ke proyek yang Anda pilih. Dashboard publik menampilkan total funding masuk, status MRV, dan dokumentasi lapangan.

**Q: Apakah saya bisa dapat sertifikat?**
Ya, sertifikat digital terbit otomatis saat pembayaran berhasil, dengan nama lengkap dan ID unik yang dapat diverifikasi.

**Q: Apa peran KKP/DKP di platform ini?**
KKP (Ditjen PSDKP) sebagai pembina formal Pokmaswas. DKP daerah sebagai validator titik proyek dan auditor pelaksanaan. ID-MAP adalah alat kerja digital, bukan pengganti otoritas.

### FAQ Internal (Tim)

**Q: Bagaimana model bisnisnya?**
Tahap pilot: gratis untuk semua. Tahap scale-up: subscription untuk korporasi (akses laporan ESG agregat) + transaction fee minimal untuk operasional. Tidak ambil dari donasi Sahabat.

**Q: Risiko terbesar?**

1. **Adopsi Pokmaswas senior yang literasi digital rendah** → mitigasi: form simpel + onboarding via DKP **+ NGO partnership (lihat Stage 11 — strategic decision)**
2. **Webhook payment gagal** → mitigasi: signature verify + idempotent + polling fallback
3. **Skeptis korporasi tentang verifikasi** → mitigasi: dashboard publik + audit trail blockchain-ready

## Stage 5 · Architecture

### Stack Decision Records

**ADR-001: Frontend → Next.js 14 (App Router)**
- **Pro:** SSR untuk SEO landing, RSC untuk data fetch tanpa API roundtrip, deployment cepat ke Vercel
- **Con:** Belajar paradigma RSC lebih kompleks dari Pages Router
- **Alternative ditolak:** Remix (less ecosystem), SvelteKit (less hire-able)

**ADR-002: Backend → Convex (Reactive DB-as-a-Service)**
- **Pro:** Live queries built-in (perfect untuk dashboard real-time), TypeScript end-to-end, mutation idempotent native
- **Con:** Vendor lock-in, lebih mahal di scale > 500K user
- **Alternative ditolak:** Supabase (perlu setup realtime layer manual), Firebase (firestore query terbatas)
- **Migration path:** Convex → Postgres+Drizzle saat user > 500K

**ADR-003: Auth → Custom session + bcrypt** (planned)
- **Saat ini:** localStorage + plaintext password (DEBT, harus dimigrate sebelum prod)
- **Target:** Convex Auth atau custom JWT + httpOnly cookie + bcrypt hash

**ADR-004: Payment → Mayar.id**
- **Pro:** QRIS coverage 55jt user Indonesia, sandbox + production di base URL berbeda, webhook standar
- **Con:** Documentation kadang kurang, fee per-tx
- **Alternative dipertimbangkan:** Midtrans (lebih enterprise tapi setup ribet), Xendit

**ADR-005: Hosting → Vercel + Cloudflare di depan**
- **Pro:** Vercel optimal untuk Next.js, Cloudflare gratis WAF + DDoS, edge caching
- **Con:** Vercel mahal di skala besar (10M+ visitor)
- **Migration path:** Vercel → self-host Coolify di VPS Indonesia + Cloudflare Pages saat diperlukan

**ADR-006: Image storage → Cloudinary (free tier)**
- **Pro:** Auto-optimize, CDN global, transformation API
- **Migration path:** Cloudinary → Cloudflare R2 + Bunny.net saat > 25GB bandwidth

**ADR-007: Map → Leaflet + Earth Engine iframe**
- **Pro:** OSM tiles gratis, Earth Engine punya layer mangrove pre-computed
- **Con:** EE iframe bukan native, sulit interaksi
- **Future:** Migrasi ke Mapbox + PostGIS saat butuh real-time vector tiles

- Typecheck pass
- Build pass
- Manual test desktop + mobile
- Login guard verified (redirect ke /daftar)
- Sertifikat ter-issue dengan owner = userId

## Stage 11 · Strategic Decision: NGO Layer for Pokmaswas

### Konteks

Risiko #1 di PRFAQ Tahap 4: **adopsi Pokmaswas senior literasi digital rendah**.
Mitigasi awal: form simpel + onboarding via DKP.

**Ide tambahan dari diskusi ahli (28 Mei 2026):**
> "Projek Pokmaswas di-pihak-ke-3-kan dengan NGO terpercaya yang menguasai lapangan dan core layanannya. Pelaporan jadi lebih akurat dan cepat, program tersalurkan ke Pokmas tepat sasaran."

### Analisis: Bisa, dan justru memperkuat positioning

**✅ Pro (kenapa ini ide bagus):**

1. **NGO sudah punya infrastruktur lapangan**
   YKAN, KEHATI, WWF, WCS, Pemberdayaan Pesisir Mandiri, dll punya tim teknis yang turun ke lokasi rutin. ID-MAP gak perlu duplicate operasi.

2. **NGO punya kapasitas pelatihan & pendampingan**
   Pokmaswas senior gak perlu belajar app sendiri — NGO field officer bantu input data via tablet/HP saat kunjungan rutin. Beban literasi digital pindah ke pendamping.

3. **MRV jadi lebih kredibel di mata juri/donor**
   NGO sudah punya methodology MRV terstandar (banyak ikut Verra/Plan Vivo/Gold Standard). Data input dari NGO = audit-grade lebih kuat dari self-report Pokmaswas.

4. **Solves chicken-and-egg di rural area**
   Kalau Pokmaswas di Pulau Buton harus pakai aplikasi sendiri, adopsi lambat. Dengan NGO sebagai intermediary, ID-MAP bisa scale ke daerah remote tanpa fisik on-the-ground tim.

5. **Cocok dengan POJK 51/2017 dan ESG corporate**
   Korporasi yang buy-in (Pertamina, Astra, dll) lebih nyaman lihat NGO partner di rantai validasi. Reduce reputational risk.

6. **Ada precedent yang sudah jalan**
   YKAN-PT Pertamina (Mangrove for Coastal Resilience), KEHATI-Bank Mandiri (Hijau Lestari), WWF-Astra International. Model "korporasi → NGO → komunitas" sudah accepted di Indonesia.

**⚠ Con / risk yang harus di-mitigate:**

1. **Risiko intermediary fee membengkak**
   NGO biasanya minta 15-25% dari dana program untuk overhead. Kalau ID-MAP janji "100% ke proyek", harus jelas: NGO fee dipotong dari corporate sponsorship layer, bukan dari donasi Sahabat.

2. **NGO kapasitas tidak merata**
   Tier-1 (YKAN, KEHATI) bagus. Tier-2-3 banyak yang sebatas reporting copy-paste. Perlu **NGO whitelist + audit periodik** dari ID-MAP.

3. **Politik & relasi NGO-Pokmaswas**
   Ada Pokmaswas yang udah terikat NGO tertentu, ada yang anti-NGO (preferensi lapor langsung ke DKP). Solusi: NGO partnership **opt-in per proyek**, bukan mandatory.

4. **Risiko gatekeeping**
   Kalau NGO jadi satu-satunya pintu, ID-MAP kehilangan diferensiasi "bottom-up dari Pokmaswas". Harus tetap buka jalur **direct Pokmaswas → ID-MAP** untuk yang capable.

5. **Liability data**
   NGO field officer input data atas nama Pokmaswas → siapa accountable kalau data salah? Perlu MoU jelas di onboarding NGO.

### Usulan Model: 3-Tier Partnership

**Tier A — Direct Pokmaswas** (eksisting)
- Pokmaswas yang sudah literate digital input langsung ke ID-MAP
- Verifikator DKP audit
- Cocok untuk Pokmaswas urban/semi-urban di Jawa

**Tier B — NGO-Mediated Pokmaswas** (proposed, baru)
- NGO whitelisted (YKAN, KEHATI, WWF, WCS, dll) jadi field facilitator
- NGO field officer input data MRV via account dengan role baru: **`mitra_facilitator`**
- Pokmaswas pemilik proyek tetap "owner" di sistem, NGO sebagai "co-operator"
- Cocok untuk Pokmaswas di daerah remote / senior

**Tier C — Corporate-Sponsored Program** (existing arah corporate)
- Korporasi beli paket "NGO + Pokmaswas + ID-MAP integration"
- ID-MAP kasih dashboard ESG agregat + audit trail
- NGO fee ditanggung corporate sponsor

## Stage 12 · Retrospective (Pre-Submission)

### What Worked

- **Convex live queries** = killer feature. Verifikator edit hero → homepage update real-time tanpa redeploy
- **3D icon dari Microsoft Fluent** — 3 file ~80KB total, looks premium tanpa Spline/Three.js
- **Schema consolidation di akhir** lebih cepat dari design upfront — biarkan organic dulu, konsolidasi saat butuh
- **Quick-dev style** untuk fitur kecil (FAQ editor, About editor) lebih efisien dari full BMAD cycle
- **Repositioning early enough** — sebelum submission, brand sudah aligned ke "Integrasi Data dan Manajemen Pesisir"

### What Didn't

- **Plaintext password** kena debt sampai akhir. Harusnya bcrypt dari Sprint 1
- **Token color** drift sampai 12 fill colors. Harusnya design tokens di-set di Sprint 0
- **`heroImageStore` IndexedDB** — legacy yang gak dibersihkan
- **Carbon credit framing awal** — sempat di-bake ke /tentang dan /faq, butuh rewrite saat repositioning

### What We'd Change Next Time

1. **Design tokens before code** — lock di tailwind.config.ts sebelum nulis komponen pertama
2. **Auth + security at S1** — bukan di S11
3. **CMS layer earlier** — verifikator-editable content harus ada dari S2
4. **Pitch validation early** — PRFAQ harus reviewed actual stakeholder (DKP) sebelum dev mulai
5. **NGO partnership di PRFAQ awal** — bukan ide afterthought, harus jadi pillar strategy dari awal
