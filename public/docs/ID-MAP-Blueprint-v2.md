## Stage 1 ┬╖ Domain Research

### Pertanyaan kunci yang dijawab

**Q: Apa itu Pokmaswas dan kenapa jadi pivot user persona utama?**

Pokmaswas (Kelompok Masyarakat Pengawas) adalah kelompok masyarakat pesisir resmi binaan **Ditjen PSDKP тАФ Kementerian Kelautan dan Perikanan**.

- Sudah punya **SK pengukuhan formal** (artinya: identitas legal sudah ada, tidak perlu KYC ulang)
- Terlatih dalam pengawasan sumber daya laut
- Tersebar **┬▒2.000 kelompok aktif** di seluruh Indonesia
- Saat ini lapor manual via WhatsApp/kertas (pain point digitalisasi nyata)

**Q: Regulasi pesisir Indonesia yang relevan?**

- **PP 27/2021** тАФ Penyelenggaraan Bidang Kelautan dan Perikanan
- **PP 5/2021** тАФ Pelaksanaan Perizinan Berbasis Risiko (RZWP3K)
- **Permen KP 36/2023** тАФ Pemberdayaan Pokmaswas
- **POJK 51/2017** тАФ Penerapan Keuangan Berkelanjutan (untuk Corporate ESG)
- **UU PDP No. 27/2022** тАФ Perlindungan Data Pribadi (relevan untuk donor data)
- **SRN-PPI** (Sistem Registri Nasional - PPI) тАФ registry karbon nasional, tetap relevan tapi bukan primary value prop

**Q: Apa yang berubah dari positioning lama?**

| Sebelum | Sesudah |
|---|---|
| Carbon credit infrastructure | Coastal data integration platform |
| KLHK (PP 98/2021) sebagai regulator | KKP/PSDKP sebagai pembina utama |
| Mangrove only | Mangrove + abrasi + penyu + ekosistem pesisir |
| B2B focus (perusahaan beli kredit) | Multi-actor (Sahabat + Mitra + DKP + Corporate) |
| Pre-market carbon = niche | Public good integration = wider reach |

**Q: Kompetitor & lansekap?**

- **Indikator (BRGM)** тАФ pemerintah, top-down monitoring mangrove
- **Tracart, Fairatmos** тАФ private carbon registry, B2B-only
- **Asuh Pesisir, KKP One Data** тАФ internal pemerintah
- **Gap yang ID-MAP isi:** bottom-up Pokmaswas + transparent donor + multi-stakeholder marketplace

## Stage 2 ┬╖ Product Brief

### Vision Statement

> "Menjadi platform terpercaya yang mempercepat pemulihan dan keberlanjutan ekosistem pesisir nusantara melalui kolaborasi data, komunitas, dan teknologi."

### Mission Statement

> "Mengintegrasikan data dan tata kelola pesisir Indonesia melalui enam layanan inti тАФ rehabilitasi mangrove, penyulaman, MRV digital, decarbonisasi aquaculture, perbaikan habitat penyu, dan pemberdayaan masyarakat pesisir тАФ secara transparan, partisipatif, dan berkelanjutan."

### Target Users (Primary)

1. **Pokmaswas / Mitra** (B2B-G operator) тАФ ┬▒2.000 kelompok ber-SK PSDKP, fokus Jatim тЖТ Jateng тЖТ Sulsel тЖТ NTT
2. **Sahabat Pesisir** (B2C donor) тАФ GenZ-Millennial 22-40 tahun di Jabodetabek-Surabaya-Bandung, donasi Rp 10rb-500rb
3. **DKP Provinsi/Kabupaten** (G2G audit) тАФ perpanjangan tangan KKP di lapangan
4. **Korporasi ESG** (B2B sponsor) тАФ emiten BEI wajib lapor POJK 51/2017
5. **NGO Konservasi** (mitra strategis) тАФ YKAN, WWF, KEHATI

### Value Proposition Triangle

- Untuk **Pokmaswas**: alat kerja digital тЖТ laporan cepat тЖТ honor lebih cepat тЖТ pengakuan
- Untuk **Sahabat**: bukti realisasi donasi тЖТ sertifikat shareable тЖТ impact nyata
- Untuk **DKP**: dashboard agregat lintas kecamatan тЖТ laporan ke pusat lebih mudah

### 6 Layanan Inti (cores)

1. Rehabilitasi Mangrove
2. Penyulaman Mangrove
3. Jasa Pemantauan Monev (MRV) Mangrove
4. Decarbonisasi Aquaculture
5. Perbaikan Habitat Penyu
6. Pemberdayaan Masyarakat Pesisir

### Differentiator

- Bottom-up dari Pokmaswas, bukan top-down dari KLHK/BRGM
- Multi-actor marketplace, bukan registry tertutup
- Live MRV dengan QRIS donation loop yang langsung tersalurkan ke kelompok pelaksana
- Future-ready: data MRV kompatibel dengan SRN-PPI saat regulasi blue carbon matang

### FAQ External

**Q: Apa beda ID-MAP dengan platform donasi lingkungan lain?**
ID-MAP fokus eksklusif pada ekosistem pesisir Indonesia dan terhubung langsung ke Pokmaswas resmi binaan KKP. Setiap proyek divalidasi DKP daerah dan dapat diaudit publik.

**Q: Bagaimana donasi saya tersalurkan?**
100% dana donasi masuk ke proyek yang Anda pilih. Dashboard publik menampilkan total funding masuk, status MRV, dan dokumentasi lapangan.

**Q: Apakah saya bisa dapat sertifikat?**
Ya, sertifikat digital terbit otomatis saat pembayaran berhasil, dengan nama lengkap dan ID unik yang dapat diverifikasi.

**Q: Apa peran KKP/DKP di platform ini?**
KKP (Ditjen PSDKP) sebagai pembina formal Pokmaswas. DKP daerah sebagai validator titik proyek dan auditor pelaksanaan. ID-MAP adalah alat kerja digital, bukan pengganti otoritas.

### FAQ Internal (Tim)

**Q: Bagaimana model bisnisnya?**
Tahap pilot: gratis untuk semua. Tahap scale-up: subscription untuk korporasi (akses laporan ESG agregat) + transaction fee minimal untuk operasional. Tidak ambil dari donasi Sahabat.

**Q: Risiko terbesar?**

1. **Adopsi Pokmaswas senior yang literasi digital rendah** тЖТ mitigasi: form simpel + onboarding via DKP **+ NGO partnership (lihat Stage 11 тАФ strategic decision)**
2. **Webhook payment gagal** тЖТ mitigasi: signature verify + idempotent + polling fallback
3. **Skeptis korporasi tentang verifikasi** тЖТ mitigasi: dashboard publik + audit trail blockchain-ready

## Stage 5 ┬╖ Architecture

### Stack Decision Records

**ADR-001: Frontend тЖТ Next.js 14 (App Router)**
- **Pro:** SSR untuk SEO landing, RSC untuk data fetch tanpa API roundtrip, deployment cepat ke Vercel
- **Con:** Belajar paradigma RSC lebih kompleks dari Pages Router
- **Alternative ditolak:** Remix (less ecosystem), SvelteKit (less hire-able)

**ADR-002: Backend тЖТ Convex (Reactive DB-as-a-Service)**
- **Pro:** Live queries built-in (perfect untuk dashboard real-time), TypeScript end-to-end, mutation idempotent native
- **Con:** Vendor lock-in, lebih mahal di scale > 500K user
- **Alternative ditolak:** Supabase (perlu setup realtime layer manual), Firebase (firestore query terbatas)
- **Migration path:** Convex тЖТ Postgres+Drizzle saat user > 500K

**ADR-003: Auth тЖТ Custom session + bcrypt** (planned)
- **Saat ini:** localStorage + plaintext password (DEBT, harus dimigrate sebelum prod)
- **Target:** Convex Auth atau custom JWT + httpOnly cookie + bcrypt hash

**ADR-004: Payment тЖТ Mayar.id**
- **Pro:** QRIS coverage 55jt user Indonesia, sandbox + production di base URL berbeda, webhook standar
- **Con:** Documentation kadang kurang, fee per-tx
- **Alternative dipertimbangkan:** Midtrans (lebih enterprise tapi setup ribet), Xendit

**ADR-005: Hosting тЖТ Vercel + Cloudflare di depan**
- **Pro:** Vercel optimal untuk Next.js, Cloudflare gratis WAF + DDoS, edge caching
- **Con:** Vercel mahal di skala besar (10M+ visitor)
- **Migration path:** Vercel тЖТ self-host Coolify di VPS Indonesia + Cloudflare Pages saat diperlukan

**ADR-006: Image storage тЖТ Cloudinary (free tier)**
- **Pro:** Auto-optimize, CDN global, transformation API
- **Migration path:** Cloudinary тЖТ Cloudflare R2 + Bunny.net saat > 25GB bandwidth

**ADR-007: Map тЖТ Leaflet + Earth Engine iframe**
- **Pro:** OSM tiles gratis, Earth Engine punya layer mangrove pre-computed
- **Con:** EE iframe bukan native, sulit interaksi
- **Future:** Migrasi ke Mapbox + PostGIS saat butuh real-time vector tiles

## Stage 8 ┬╖ Dev Story Sample

**Story: US-S02 (Sahabat Donasi QRIS)**

**Context for dev:**
- File: `src/app/donasi-cepat/[projectId]/page.tsx`, `src/app/api/payment/create-qris/route.ts`, `convex/contributions.ts`
- Convention: Convex mutations harus idempotent, gunakan `withIndex` bukan `filter`, validator di schema.ts
- Style: 2-column desktop, stacked mobile, brand emerald-600 sebagai primer
- Edge case: project status null (tampilkan "Proyek tidak ditemukan"), webhook gagal (polling fallback 3s)

**Tasks:**
1. UI: form pilih nominal preset + custom input
2. API: POST `/api/payment/create-qris` panggil Mayar atau dummy
3. State machine: idle тЖТ generating тЖТ waiting тЖТ paid (atau error)
4. Polling: status check tiap 3s sampai paid
5. Convex: `createPending` тЖТ `attachPaymentId` тЖТ `confirmByPaymentId` (auto-issue cert)
6. Test: buat dummy contribution, simulate payment, cek cert ter-insert

## Stage 11 ┬╖ Strategic Decision: NGO Layer for Pokmaswas

### Konteks

Risiko #1 di PRFAQ Tahap 4: **adopsi Pokmaswas senior literasi digital rendah**.
Mitigasi awal: form simpel + onboarding via DKP.

**Ide tambahan dari diskusi ahli (28 Mei 2026):**
> "Projek Pokmaswas di-pihak-ke-3-kan dengan NGO terpercaya yang menguasai lapangan dan core layanannya. Pelaporan jadi lebih akurat dan cepat, program tersalurkan ke Pokmas tepat sasaran."

### Analisis: Bisa, dan justru memperkuat positioning

**тЬЕ Pro (kenapa ini ide bagus):**

1. **NGO sudah punya infrastruktur lapangan**
   YKAN, KEHATI, WWF, WCS, Pemberdayaan Pesisir Mandiri, dll punya tim teknis yang turun ke lokasi rutin. ID-MAP gak perlu duplicate operasi.

2. **NGO punya kapasitas pelatihan & pendampingan**
   Pokmaswas senior gak perlu belajar app sendiri тАФ NGO field officer bantu input data via tablet/HP saat kunjungan rutin. Beban literasi digital pindah ke pendamping.

3. **MRV jadi lebih kredibel di mata juri/donor**
   NGO sudah punya methodology MRV terstandar (banyak ikut Verra/Plan Vivo/Gold Standard). Data input dari NGO = audit-grade lebih kuat dari self-report Pokmaswas.

4. **Solves chicken-and-egg di rural area**
   Kalau Pokmaswas di Pulau Buton harus pakai aplikasi sendiri, adopsi lambat. Dengan NGO sebagai intermediary, ID-MAP bisa scale ke daerah remote tanpa fisik on-the-ground tim.

5. **Cocok dengan POJK 51/2017 dan ESG corporate**
   Korporasi yang buy-in (Pertamina, Astra, dll) lebih nyaman lihat NGO partner di rantai validasi. Reduce reputational risk.

6. **Ada precedent yang sudah jalan**
   YKAN-PT Pertamina (Mangrove for Coastal Resilience), KEHATI-Bank Mandiri (Hijau Lestari), WWF-Astra International. Model "korporasi тЖТ NGO тЖТ komunitas" sudah accepted di Indonesia.

**тЪа Con / risk yang harus di-mitigate:**

1. **Risiko intermediary fee membengkak**
   NGO biasanya minta 15-25% dari dana program untuk overhead. Kalau ID-MAP janji "100% ke proyek", harus jelas: NGO fee dipotong dari corporate sponsorship layer, bukan dari donasi Sahabat.

2. **NGO kapasitas tidak merata**
   Tier-1 (YKAN, KEHATI) bagus. Tier-2-3 banyak yang sebatas reporting copy-paste. Perlu **NGO whitelist + audit periodik** dari ID-MAP.

3. **Politik & relasi NGO-Pokmaswas**
   Ada Pokmaswas yang udah terikat NGO tertentu, ada yang anti-NGO (preferensi lapor langsung ke DKP). Solusi: NGO partnership **opt-in per proyek**, bukan mandatory.

4. **Risiko gatekeeping**
   Kalau NGO jadi satu-satunya pintu, ID-MAP kehilangan diferensiasi "bottom-up dari Pokmaswas". Harus tetap buka jalur **direct Pokmaswas тЖТ ID-MAP** untuk yang capable.

5. **Liability data**
   NGO field officer input data atas nama Pokmaswas тЖТ siapa accountable kalau data salah? Perlu MoU jelas di onboarding NGO.

### Usulan Model: 3-Tier Partnership

**Tier A тАФ Direct Pokmaswas** (eksisting)
- Pokmaswas yang sudah literate digital input langsung ke ID-MAP
- Verifikator DKP audit
- Cocok untuk Pokmaswas urban/semi-urban di Jawa

**Tier B тАФ NGO-Mediated Pokmaswas** (proposed, baru)
- NGO whitelisted (YKAN, KEHATI, WWF, WCS, dll) jadi field facilitator
- NGO field officer input data MRV via account dengan role baru: **`mitra_facilitator`**
- Pokmaswas pemilik proyek tetap "owner" di sistem, NGO sebagai "co-operator"
- Cocok untuk Pokmaswas di daerah remote / senior

**Tier C тАФ Corporate-Sponsored Program** (existing arah corporate)
- Korporasi beli paket "NGO + Pokmaswas + ID-MAP integration"
- ID-MAP kasih dashboard ESG agregat + audit trail
- NGO fee ditanggung corporate sponsor

## Stage 13 ┬╖ Dampak Keberlanjutan (SDGs) & Visi Global

### Penyelarasan dengan Sustainable Development Goals (SDGs)

ID-MAP bukan sekadar platform donasi тАФ sebagai platform **keberlanjutan ekosistem pesisir**, keenam layanan inti dirancang selaras dengan agenda **Sustainable Development Goals (SDGs)** PBB, terutama pilar lingkungan, iklim, dan kesejahteraan masyarakat pesisir.

**Pemetaan 6 Layanan Inti тЖТ SDGs:**

| Layanan Inti | SDGs Terkait |
|---|---|
| 1. Rehabilitasi Mangrove | SDG 13 (Iklim) ┬╖ SDG 14 (Ekosistem Laut) ┬╖ SDG 15 (Ekosistem Darat) |
| 2. Penyulaman Mangrove | SDG 13 ┬╖ SDG 14 ┬╖ SDG 15 |
| 3. Jasa Pemantauan Monev (MRV) | SDG 13 ┬╖ SDG 14 ┬╖ SDG 17 (Kemitraan & transparansi data) |
| 4. Decarbonisasi Aquaculture | SDG 12 (Produksi Bertanggung Jawab) ┬╖ SDG 13 ┬╖ SDG 14 |
| 5. Perbaikan Habitat Penyu | SDG 14 ┬╖ SDG 15 |
| 6. Pemberdayaan Masyarakat Pesisir | SDG 1 (Tanpa Kemiskinan) ┬╖ SDG 8 (Pekerjaan Layak) ┬╖ SDG 10 (Mengurangi Ketimpangan) ┬╖ SDG 11 (Komunitas Berkelanjutan) |

**SDGs inti yang diusung ID-MAP:**

- **SDG 14 тАФ Ekosistem Lautan (Life Below Water):** poros utama. Mangrove, penyu, dan rehabilitasi pesisir adalah inti misi platform.
- **SDG 13 тАФ Penanganan Perubahan Iklim:** mangrove sebagai *blue carbon sink*, decarbonisasi aquaculture menurunkan emisi.
- **SDG 15 тАФ Ekosistem Daratan:** vegetasi pesisir dan habitat sebagai zona transisi darat-laut.
- **SDG 1 & 8 тАФ Tanpa Kemiskinan & Pekerjaan Layak:** honor Pokmaswas tersalurkan cepat, pesisir jadi sumber penghidupan berkelanjutan.
- **SDG 12 тАФ Konsumsi & Produksi Bertanggung Jawab:** aquaculture rendah karbon.
- **SDG 17 тАФ Kemitraan untuk Tujuan:** model multi-aktor (Sahabat + Pokmaswas + DKP + NGO + Korporasi) adalah perwujudan langsung dari kemitraan lintas sektor.

> Penyelarasan SDGs ini memperkuat narasi ke juri Tahap 2, donor publik, dan korporasi ESG (POJK 51/2017) yang laporan keberlanjutannya wajib memetakan kontribusi terhadap SDGs.

### Menunjang Ekosistem QRIS Bank Indonesia hingga ke Pesisir

Donasi mikro ID-MAP berjalan di atas **QRIS** тАФ standar pembayaran nasional Bank Indonesia. Dengan menyalurkan donasi Sahabat Pesisir ke proyek Pokmaswas di daerah pesisir dan remote, ID-MAP turut **memperluas penetrasi ekosistem QRIS BI hingga menyentuh komunitas pesisir** yang selama ini berada di luar jangkauan layanan keuangan digital.

- **Inklusi keuangan pesisir:** transaksi digital masuk ke kelompok masyarakat yang sebelumnya cash-based / unbanked.
- **Sejalan dengan misi BI** untuk perluasan akseptasi QRIS ke seluruh lapisan masyarakat dan wilayah 3T.
- **Setiap donasi = jejak transaksi QRIS baru** di wilayah pesisir, memperkaya ekosistem pembayaran nasional.

### Visi Global: QRIS Cross-Border ASEAN + Mode Bahasa Inggris

Bank Indonesia telah meluncurkan **QRIS Cross-Border** dengan negara-negara ASEAN (Thailand, Malaysia, Singapura) sebagai bagian dari **Regional Payment Connectivity (RPC)**, dan terus diperluas. Ini membuka jalan agar donasi lintas negara dapat masuk ke proyek pesisir Indonesia tanpa friksi konversi.

ID-MAP **future-ready** untuk peluang ini:

- **Mode Bahasa Inggris sudah tersedia** di web app (toggle ID тЖФ EN via `LanguageContext`), sehingga donatur internasional, diaspora Indonesia, dan mitra global dapat menggunakan platform secara penuh.
- **QRIS cross-border** memungkinkan Sahabat Pesisir dari negara ASEAN berdonasi langsung memakai aplikasi pembayaran negaranya тАФ menjangkau pasar donor regional.
- **Kombinasi mode EN + QRIS cross-border** memposisikan ID-MAP sebagai jembatan donasi konservasi pesisir Indonesia ke skala Asia, bukan hanya domestik.

> Jangka panjang: ID-MAP dapat menjadi etalase konservasi pesisir Indonesia bagi donor & korporasi ASEAN, menumpang infrastruktur QRIS cross-border BI dan didukung antarmuka multibahasa.
